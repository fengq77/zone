SWC To Machine Mapping Editor

> Software Component：显示选定的 Adaptive Software Components。

> Process：显示分配给软件组件的进程。每一个映射到 Machine 的软件组件需要一个进程来运行，因此

与映射一起，将自动创建一个流程并将其链接到软件组件。

> Software To Hardware Mapping：显示连接与机器一起使用的软件组件。在 AUTOSAR Adaptive 中，这个映射是称为 Process to Machine Mapping。

> Machine：显示映射到软件组件的Process Units。通过下拉列表选择 Machine。然后是一个SW Component Mapping 和一个Process被创建。

> Shall Run On：通过下拉列表选择进程应运行的核心。

> Shall Not Run On：通过下拉列表选择进程不应运行的核心。

一旦软件架构和拓扑被建模，软件组件可以映射到拓扑中的机器。

使用SWC To Machine Mapping Editor，可以将软件映射到硬件，以创建必要的流程每个软件组件以及将每个进程分配给一个核心微处理器。

注意：对于一个进程，只能设置Shall Run On 或Shall Not Run On
