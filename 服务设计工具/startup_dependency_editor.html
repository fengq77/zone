Service Instance

启动依赖项编辑器提供以下列：

Process：显示由软件到硬件映射创建的进程。

Startup Dependency：显示进程的模式启动依赖项。通过表格单元格中的"+"按钮创建。此工件是对启动配置的引用

这要么取决于机器、过程的模式，要么被考虑独立。

Function Group Dependency

Function Groups：要定义依赖于功能组的启动配置，必须引用功能组。可用模式分组在一个模式中声明组。通过下拉列表选择模式声明组。

Mode：通过下拉列表选择特定模式。

Startup Configuration

StartupConfiguration：通过定义独立的启动配置表格单元格中的按钮。

Scheduling Policy：通过下拉列表选择启动配置的调度策略。

Scheduling Priority：输入启动配置的调度优先级。

Option：通过以下方式创建启动选项作为启动配置的子工件表格单元格中的按钮。您还可以在此处编辑启动选项的名称。在AUTOSAR，名称将导出为 optionName。

Kind：输入启动选项的种类。

Value：输入启动选项的值。

在 AUTOSAR Adaptive Explorer 的 Application Deployment 类别中，选择一个类别下的Software Deployment Package显示Startup Dependency Editor
