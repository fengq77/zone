服务实例代表服务接口（Service Interfaces）通信层。 在 PREEvision 中，它们由信号路由器自动创建。

此外，信号路由器计算服务提供商和当前网络拓扑中的消费者。

这些在 AUTOSAR 导出中不会考虑辅助工件。

途径一、

信号路由器可以在 SWC to Machine Mapping Editor 中启动：

1. 关于AUTOSAR Adaptive的Adaptive Software Architecture类

在资源管理器中，选择 SWC to Machine Mapping Editor。

2. 按表工具栏中的        按钮。

途径二、

在菜单栏上下文中选择Root Composition右键选择Run Signal Router…

在这种情况下，信号路由器将根据配置文件在没有用户交互的情况下运行。

也可以在 Root Composition、特定软件上启动信号路由器组件或通过上下文菜单在端口上。信号路由器产生以下工件：

> 在通讯包中（Communication Package）：

> 每个以太网集群一个以太网通信集群

> 每个总线连接器一个以太网控制器配置

> 服务实例：每个端口一个，具有分配的服务接口和数据通过总线传输。

> 为以太网集群中的每个交换机配置一个交换机配置

> 一个默认 VLAN，每个总线连接器包含一个以太网 ECU 接口

> 描述通信的信号传输和 PDU 传输小路

> 一个网络端点包，其中包含描述网络端点的网络端点以太网连接到总线系统

> 一个套接字适配器配置，每个 ECU 包含一个套接字地址界面。此外，每个服务提供者创建一个套接字连接包它引用属于服务提供者的套接字地址和包含引用属于的套接字地址的套接字连接服务的各自消费者

>在布局包中（Layout Package）：

> 对于每个信号传输，信号路由器创建一个信号，一个系统信号和每个信号的一个信号 IPDU。这些神器只是辅助

用于信号路由器创建和更新模型的工件。

> 映射：

> 端口传输映射（Port Transmission Mappings）：在基于信号的通信中，这些创建映射以将每个信号传输与发送或软件的接收端口。因此，可以区分不同的信号由不同的端口发送。

> 事件到信号映射，方法到信号映射（Event to Signal Mappings, Method to Signal Mappings）：这些工件映射系统信号的服务接口元素。每个方法，映射到两个系统信号。每一次活动并且属性的每个通知程序都映射到一个系统信号。这些信号路由器的操作需要映射。

> Adaptive Service Instance Port Mapping：这个映射对应于AUTOSAR 中的 Service Instance To Port Prototype Mapping，它表示服务实例与某个端口和进程之间的连接。这也是有助于区分不同的端口产品或使用相同的端口服务接口。

自适应服务实例端口映射也显示在服务实例中到端口和进程编辑器，可以在服务实例的类别中找到

AUTOSAR 自适应资源管理器。 如果不使用信号路由器来创建这些映射，可以在此编辑器中手动创建它们。

创建服务实例后，必须为总线配置它们沟通。 服务实例需要用于总线通信的参数，包括服务的 SOME/IP 标识、TP 以及时间信息服务器端的服务提供和客户端的服务搜索。

配置服务实例

SOME/IP配置

对于设置服务实例的标识符，AUTOSAR Adaptive Explorer 提供SOME/IP Service Instance Editor。 在资源管理器的服务实例类别中，选择一个 Component Package 以显示 SOME/IP Service Instance Editor
