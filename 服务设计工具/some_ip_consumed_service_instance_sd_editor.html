Service Instance

SOME/IP Consumed Service Instance SD Editor 提供以下列：

> SOME/IP Service Instance：显示消耗的 SOME/IP 服务实例包含在服务实例包中。

> General Finding Setting：

> Service ID：此属性描述所需的服务实例 ID。

> Major Version and Minor Version：这些强制参数用于配置服务发现将针对哪个版本的 SOME/IP 服务搜索。

> Find Time To Live：此属性定义服务查找的时间（以秒为单位）有效的。

> Initial Delay：初始查找阶段设置：

> Max Initial Delay：以秒为单位的最大值，以随机延迟第一个服务寻找。

> Min Initial Delay：以秒为单位的最小值，以随机延迟第一个服务寻找。

> Repetition Delay：重复等待阶段设置：

> Repetition Base Delay：服务发现重复的基本延迟。

> Max Repetitions：描述服务查找的最大重复次数。在客户端，事件处理的时间也可以使用

以下参数：

> SOME/IP Event Group Instance：显示 SOME/IP Event Group Instance服务实例。

> SOME/IP Event Group：显示相应的事件组。

> Request Response Max Delay：允许的最大响应延迟（以秒为单位）响应多播收到的请求。

> Request Response Min Delay：允许的最小响应延迟（以秒为单位）响应多播收到的请求。

> Time To Live：定义此事件的订阅时间（以秒为单位）客户所期望的。对于可用的 Method 实例，可以编辑以下设置：

> UDP Collection Trigger：定义一个Service Interface的元素是否有助于触发 UDP 数据传输，如果数据收集是启用。有效值为“ALWAYS”或“NEVER”。

> UDP Collection Buffer Timeout:最大时间，传出消息可能由于数据收集延迟。
