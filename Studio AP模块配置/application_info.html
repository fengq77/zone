ES33

列表数据

1、Application Name：App名称

2、Platform Level：是否是平台级程序

3、Reporting Status：是否上报状态；如为true，值对应REPORTS-EXECUTIOTN-STATE，如值为false，值为DOES-NOT-REPORTS-EXECUTIOTN-STATE

4、Cores Choose Behavior：选择Run还是Not Run

5、Processor Cores：选择运行的core信息，支持多选

6、Execute Path：bin文件路径

7、Restart Attempts Number：“EM启动进程失败尝试重启的次数”；

8、Function Cluster Affiliation：“功能集群从属关系”；SM必须配置为“STATE_MANAGEMENT”且必填

注：以上配置信息如引用EM配置中的数据，则在EM配置中可看到哪些APP配置

9、Shall Run On：选择运行的进程及核信息；

10、操作：修改见【修改项目信息】弹窗

               删除见【删除】弹窗

               针对App配置EM信息，见【EM配置】

             【PHM配置】，如未配置PHM模版，则不显示该按钮

             【PER配置】，如未配置PHM模版，则不显示该按钮

1、Application Name：置灰，不可修改

2、Platform Level：是否是平台级程序，下拉选项为默认为“请选择”、“true”“false”

3、Reporting Status：是否上报状态，下拉选项为默认为“请选择”、“true”“false”

4、Core Info：“选择运行的进程及（多选）核信息”；下拉选项为“EM配置”的“Processor-Core”，可以选择多个Core

1、未配置过PHM/PER，首次需要选择引入模版

2、引入模版后，直接弹窗到【app PHM配置】或【app PER配置】编辑

3、如导入的arxml含有配置信息，则默认解析显示Arxml的配置信息
